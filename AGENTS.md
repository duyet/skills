# Agents - Team Coordination Patterns

This document defines the agent coordination patterns used in @duyet/skills.

## Agent Types

### Leader Agent
- **Purpose**: Coordinates parallel workstreams across multiple agents
- **Use**: Complex multi-domain tasks requiring decomposition
- **Triggers**: Large features, refactors, multi-component work

### Senior Engineer Agent
- **Purpose**: Implements complex features with architectural decisions
- **Use**: Medium to high complexity tasks requiring design choices
- **Triggers**: API design, component architecture, performance work

### Junior Engineer Agent
- **Purpose**: Executes well-defined, straightforward tasks
- **Use**: Clear specifications, routine implementations
- **Triggers**: Simple features, bug fixes, documentation

### Duyetbot Agent
- **Purpose**: Pragmatic software development companion
- **Use**: Full-stack development with engineering discipline
- **Triggers**: Feature development, code review, troubleshooting

## Coordination Patterns

### Parallel Execution
Break independent tasks into parallel workstreams:

```
Task A (Backend)  │  Task B (Frontend)  │  Task C (Infra)
     │                  │                   │
     └──────────────────┴───────────────────┘
                      │
               Integration Point
```

### Sequential with Parallel Support
Critical path sequential, supporting work parallel:

```
Sequential:  Design → Core API → Integration
                    ↓
Parallel:        UI Docs Tests Monitoring
```

### Domain-Based Lanes
Group by architectural domain:

```
Frontend Lane     Backend Lane     Data Lane
─────────────     ─────────────    ──────────
Component A       API Endpoint     Schema
Component B       Middleware       Migration
Styles            Validation       Indexes
```

## Agent Selection Guide

| Task Complexity | Best Agent | Parallelizable |
|-----------------|------------|----------------|
| Simple, clear spec | Junior | Yes |
| Medium, some decisions | Senior | Yes |
| Large, multi-domain | Leader | N/A (orchestrates) |
| Full feature work | Duyetbot | Yes (via spawn) |
